La Problematique

Langage Naturel

Les utilisateurs expriment leurs besoins de voyage de maniere variee : "Je veux aller de Paris a Lyon", "Comment me rendre a Marseille depuis Bordeaux ?"

Extraction d'Entites (NER)

Identifier automatiquement les villes de depart et d'arrivee dans des phrases en francais, meme avec des ambiguites (prenoms = villes)

Classification Valide/Invalide

Distinguer les ordres de voyage valides des phrases sans intention de voyage : "Il fait beau" → INVALID

Sources de Donnees

Exploiter les donnees SNCF Open Data (gares, connexions) et generer un dataset d'entrainement de 10,000 phrases annotees

Architecture du Pipeline

INPUT

sentenceID, sentence (CSV UTF-8)

MODULE NLP

Preprocessing → NER → Classification (spaCy / CamemBERT)

RESOLUTION VILLE → GARE

Associe chaque ville extraite a sa gare SNCF (KD-Tree + Haversine)

MODULE PATHFINDING

Graphe SNCF → Dijkstra → Chemin optimal

OUTPUT

sentenceID, ville_dep, gare_dep, ville_arr, gare_arr, itineraire

Phrases ambigues

"Je vais a {Arrivee}" (depart manquant)

Resolution Ville → Gare & Pathfinding

Correspondance Ville → Gare

Le NLP extrait des noms de villes. Ce module associe chaque ville extraite a sa gare SNCF via code INSEE. Si la ville n'a pas de gare, on trouve la gare la plus proche.

KD-Tree + Haversine

Recherche spatiale en O(log n) pour trouver la gare la plus proche d'une commune. Distance calculee sur sphere terrestre (formule Haversine).

Graphe du Reseau SNCF

Les gares sont des noeuds, les connexions ferroviaires sont des aretes. Donnees importees depuis SNCF Open Data (CSV).

Dijkstra (Plus Court Chemin)

Algorithme classique de pathfinding. Retourne la sequence de gares : Depart → Etape1 → ... → Destination.
